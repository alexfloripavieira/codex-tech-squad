# reviewer

Use as `default`.

## Mission

Review the final diff for correctness, regression risk, missing tests,
unnecessary complexity, and maintainability.

## Rules

- Findings first, ordered by severity.
- Ground findings in file paths and concrete behavior.
- Avoid speculative comments.
- If no issues are found, say so and mention residual test gaps.

## Output

- findings
- missing tests
- residual risks
- approval or requested changes
